02
Definition

What cookies are

A cookie is a small text value that a website can ask a browser to store on a device.

Cookies can allow a website or service provider to recognize a browser, remember a preference, maintain a session or receive information about how a feature is used.

Cookies may be:

  • Session cookies, which normally expire when the browser session ends.
  • Persistent cookies, which can remain for a defined period or until removed.
  • First-party cookies, which are set by the website domain being visited.
  • Third-party cookies, which may be set by an external service included on the website.

Not every browser-stored preference is technically a cookie. Similar technologies can perform related functions.

03
Similar technologies

Browser storage and related tools

SecureHabit may use technologies that work similarly to cookies, including:

Local storage

Stores a limited value in the browser until it is removed or replaced.

Session storage

Stores information for the current browser tab or session.

Scripts

Enable website functions, interactive elements, security controls or performance features.

Server logs

Record technical requests, errors and security information at hosting or infrastructure level.

This policy uses the term “cookies” broadly where it is helpful to describe cookies and comparable website technologies together.

06
Third-party services

External resources and technical requests

The website may request resources from third-party providers to deliver features such as fonts, icons, animation libraries, hosting, security or form delivery.

When a browser requests a resource from an external provider, that provider may receive technical information such as:

  • Internet protocol address.
  • Browser and device information.
  • Requested file or page.
  • Date and time of the request.
  • Referring page or website information.

A third-party provider may use cookies or other technologies according to its own privacy policy and service terms.

Services included during development or demonstration may be replaced, self-hosted, removed or reconfigured before a production launch.

Third-party controls are separate

SecureHabit does not control every cookie, log or technical process used by an independent external provider.

08
Browser controls

Removing or blocking stored website data

Most browsers provide settings that allow you to:

  • View stored cookies and website data.
  • Delete data for one website or all websites.
  • Block third-party cookies.
  • Block all cookies.
  • Clear local storage and cached website files.
  • Use private or incognito browsing modes.

Browser settings differ by device and software version. Review the help documentation supplied by your browser provider for current instructions.

Blocking storage may affect functionality

Some website preferences, consent controls or interactive features may not operate as intended when all cookies and local storage are blocked.

09
Retention

How long browser-stored values may remain

The duration of a cookie or browser-storage value depends on its function and configuration.

A value may remain:

  • Until the current browser session ends.
  • For a defined number of days or months.
  • Until the value is replaced with a new preference.
  • Until you remove it through website or browser controls.
  • Until the browser automatically removes stored website data.

Server-side technical logs may follow different retention periods based on security, hosting, troubleshooting and legal requirements.
