  1. Display name: “Account Team” can be edited by the sender and does not confirm identity.
  2. Real sender address: account-alert@access-message.example should be compared with expected organizational domains.
  3. Reply-to address: verify-access@outside-mail.example differs from the visible sender address.
  4. Link destination: the fictional button points to training.invalid/account-review rather than an approved company service.
  5. Attachment: Account-Review.zip is unexpected and uses a compressed file type.
  6. Message tone: the message creates urgency and threatens permanent access loss.
  7. Requested action: the recipient is asked to sign in through the message instead of opening an approved service independently.

Phishing Review Checklist

A concise six-point review for unexpected messages, links, files and requests.

  1. Review the full sender address.
  2. Check the message context.
  3. Inspect links and attachments.
  4. Review the requested action.
  5. Verify independently.
  6. Follow the reporting process.
